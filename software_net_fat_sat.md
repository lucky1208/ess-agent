### 1.2 验收阶段定义决策树

```
项目阶段？
  ├── 设备出厂前 → FAT (Factory Acceptance Test)
  │   ├── 地点：制造商工厂
  │   ├── 范围：单台设备/单舱体
  │   └── 目的：确认设备符合设计规格后发运
  │
  ├── 设备到场后 → Pre-Commissioning (安装前检查)
  │   ├── 外观检查、开箱检查
  │   └── 绝缘检查、安装条件确认
  │
  ├── 安装完成后 → Commissioning (调试)
  │   ├── 单机调试（单体设备上电、通信、功能确认）
  │   └── 分系统调试（BMS+PCS+消防联动）
  │
  ├── 并网前 → SAT (Site Acceptance Test)
  │   ├── 地点：项目现场
  │   ├── 范围：全系统联合验收
  │   ├── 包含：并网测试、性能测试、保护联动、连续试运行
  │   └── 目的：确认系统满足合同和并网要求
  │
  └── 并网后 → PAC / FAC (预/最终验收)
      ├── PAC (Provisional Acceptance)：试运行168h后签发
      └── FAC (Final Acceptance)：质保期结束后签发
```

### 1.3 验收等级与判定标准

| 验收项分类 | 等级 | 判定标准 | 不合格处置 |
|-----------|------|---------|-----------|
| **安全关键项** | A级 | 必须100%合格，一票否决 | 整改至合格，不得让步接收 |
| **功能性能项** | B级 | ≥95%合格，每项偏差≤5% | 限期整改，可签条件性PAC |
| **一般检查项** | C级 | ≥90%合格，可附备注 | 记录缺陷，约定期限整改 |
| **文档资料项** | D级 | 全部齐全 | 补充完成后签字 |

## 二、FAT工厂验收测试完整规范

### 2.1 FAT整体流程

```
Step 1: 测试准备 (Day -7)
   ├── 确认出厂检验大纲（ITP - Inspection & Test Plan）
   ├── 准备测试仪器（已校准、在有效期内）
   ├── 准备测试记录表（纸质+电子双备份）
   ├── 确认测试环境（供电、通信、安全防护）
   └── 召集测试参与方（供应商、业主/监理、第三方如有）

Step 2: 外观与结构检查 (Day 1 上午)
   ├── 设备外观、铭牌、标识检查
   ├── 尺寸、重量、接口位置核对
   ├── 防护等级确认（IP等级、防腐等级）
   └── 运输状态检查（绑扎固定、防震垫）

Step 3: 布线/电气连接检查 (Day 1 下午)
   ├── 线缆走向和固定检查
   ├── 端子紧固力矩抽查（10%端子）
   ├── 绝缘电阻测试（主回路、二次回路）
   └── 接地连续性测试

Step 4: 上电检查 (Day 2)
   ├── 辅助电源上电（24V/220V）
   ├── 控制系统上电（BMS/EMS/PCS控制器）
   ├── 电源电压/纹波测量
   └── 上电时序验证

Step 5: 通信功能测试 (Day 2-3)
   ├── 内部通信检查（CAN/RS485/Ethernet）
   ├── 协议一致性（寄存器映射核对）
   ├── 通信冗余切换测试
   └── 通信异常恢复测试

Step 6: 保护功能测试 (Day 3-4)
   ├── 过压/欠压保护（电芯级、模组级、簇级、系统级）
   ├── 过温保护（逐级验证）
   ├── 绝缘故障保护
   ├── 紧急停机测试
   └── 消防联动测试（模拟）

Step 7: 性能测试 (Day 4-5)
   ├── 充放电功能验证
   ├── 恒功率精度测试
   ├── SOC标定验证
   ├── 效率测试
   └── 容量测试（可选择抽测）

Step 8: 出厂报告与签字 (Day 5)
   ├── 汇总所有测试数据
   ├── 填写不合格项记录(NCR)
   ├── 签署FAT报告
   └── 签发放行单(Release Note)
```

## 三、SAT现场验收测试完整规范

### 3.1 SAT整体流程

```
Step 1: 安装性检查 (Pre-Commissioning)
   ├── 设备外观/基础/安装质量
   ├── 接地系统验收
   ├── 电缆敷设验收
   └── 消防系统验收

Step 2: 单体设备调试
   ├── 变压器/开关柜交接试验
   ├── PCS单机调试
   ├── BESS上电激活
   ├── EMS系统启动
   ├── 空调/液冷系统调试
   └── 消防系统功能验证

Step 3: 系统联调
   ├── 通信全链路打通
   ├── BMS-PCS联调
   ├── EMS全系统控制功能
   ├── AGC/AVC功能联调（如有）
   └── 一次调频功能测试（如有）

Step 4: 保护联动测试
   ├── 全系统急停测试
   ├── 消防联动停机测试（必须执行）
   ├── 防孤岛保护测试
   ├── 全保护矩阵验证
   └── 告警信号核对

Step 5: 并网测试
   ├── 并网点电能质量测试
   ├── 充放电转换测试
   ├── 满功率充放电测试
   ├── 功率控制精度测试(AGC)
   └── 电网适应性测试

Step 6: 性能测试
   ├── 充放电效率测试
   ├── 响应时间测试
   ├── 容量测试（可选择）
   ├── SOC精度验证
   └── 辅助功耗测试

Step 7: 连续试运行 (168h)
   ├── 无故障运行168小时
   ├── 含至少2个完整充放电循环
   └── 记录全时段运行数据

Step 8: 验收签字
   ├── 汇总全部测试数据
   ├── 提交验收报告
   └── 业主/监理/供应商三方签字
```

### 3.2 SAT安装性检查清单

### 3.4 SAT保护联动测试

#### 3.4.1 消防联动停机测试（SAT中最关键的A级安全项）

```
【测试前提】
  - 消防系统已完成自身功能验收
  - 所有断路器处于合闸状态
  - 系统处于运行状态（模拟状态，实际电池未投入大功率）

【测试步骤】
  1. 确认所有人员安全站位
  2. 手动触发消防主机告警信号（烟感+温感同时触发/手动报警按钮）
  3. 用录波装置同时记录以下信号：
     - 消防主机输出干接点变位时刻
     - 直流侧断路器分闸线圈得电时刻
     - 交流侧断路器分闸线圈得电时刻
     - 断路器辅助触点变位时刻
     - PCS停机状态变位时刻
     - BMS告警信号变位时刻
  4. 检查以下设备是否全部执行停机动作

【验收标准】
  消防告警→直流断路器分闸完成： ≤1s
  消防告警→交流断路器分闸完成： ≤1s
  消防告警→PCS停机完成：       ≤500ms
  消防告警→BMS紧急告警置位：    ≤500ms
  所有停机回路：                硬接线动作，不得依赖软件/通信

【必须验证】：
  □ 消防停机回路是否为硬接线（从消防主机干接点→安全继电器→分闸线圈）
  □ 通信中断情况下（拔除消防主机到EMS通信线），手动触发消防告警，系统能否正常执行紧急停机
  □ 消防停机后，是否禁止自动重合闸（需人工复位确认）
```

#### 3.4.2 防孤岛保护测试

```
【测试方法 - RLC谐振法】
  前提：配置RLC并联负载模拟本地负荷
  1. 将RLC负载调至PCS输出功率匹配状态（Q=1的谐振条件）
  2. 断开PCC断路器（模拟电网停电）
  3. 记录PCS检测到孤岛的时间
  4. 记录PCS停机的时间

【判定标准】
  - 孤岛检测时间 ≤ 2s (GB/T 36548)
  - PCS必须在检测到孤岛后200ms内停止输出
  - PCS停机后必须锁定（禁止自动重启）
  - PCC断路器断开后，防孤岛装置必须在规定时间内动作

【测试点】：
  在25%, 50%, 75%, 100%额定功率各测1次
```

#### 3.4.3 全系统急停测试（ESD）

```
【测试步骤】
  1. 系统处于正常运行状态（或模拟运行状态）
  2. 按下主控室急停按钮
  3. 记录：急停按下→全系统断电的时间
  4. 检查所有断路器是否分闸
  5. 复位急停按钮，手动逐一恢复系统

【判定标准】
  - 从按下到所有主回路断开：≤500ms
  - 按下一个急停按钮，所有分布式急停回路均响应
  - 急停回路为冗余独立硬接线系统（双重冗余）

【验证点】：
  □ 主控室急停按钮
  □ 集装箱（电池舱）急停按钮
  □ PCS柜急停按钮
  □ 每个急停按钮均能独立触发全系统停机
```

## 七、性能测试规范

### 7.1 充放电效率测试

```
【测试目的】验证PCS及系统整体的充放电效率是否满足合同/标准要求

【测试方法 - 全功率点法】
  1. 在25%, 50%, 75%, 100%额定功率点各执行1次完整充放电循环
  2. 使用0.2级(或0.5级)功率分析仪在交流侧和直流侧独立测量
  3. 记录每个功率点的充电效率和放电效率
  4. 计算PCS加权效率：
     加权效率 = Σ(效率_i × 功率权重_i)

【测试方法 - 额定功率连续法】
  1. 在额定功率下连续执行3次完整充放电循环
  2. 取3次循环的平均效率
  3. 对比合同约定的效率保证值

【数据采集要求】
  - 采样周期：≤1s
  - 使用功率分析仪同步采集AC和DC侧功率
  - 辅助功耗需单独计量（用独立电表记录站用电能）
  
【判定标准】
  - PCS最大效率：≥98.5%（或合同约定值）
  - PCS加权效率：≥98%（或合同约定值）
  - 系统综合效率：≥85%（含辅助功耗）
```

### 7.2 响应时间测试

```
【测试目的】验证系统对调度/EMS指令的响应速度

【有功功率响应时间测试】
  条件：系统处于并网运行稳定状态
  方法：
    1. 使用高速录波装置同步记录PCS功率输出和指令信号
    2. 在稳定运行时下发功率阶跃指令(0→50%Pn, 50%→100%Pn等)
    3. 记录从指令发出到功率达到90%目标值的时间
    4. 在10个不同功率点重复测试，取最大值
  
  判定：
    - 有功功率响应时间：≤500ms（一次调频），≤2s（AGC调节）
    - 超调量：≤10%Pn
    - 稳态误差：≤±2%Pn

【充放电切换响应时间测试】
  方法：
    1. 以50%Pn充电，稳定运行
    2. 发指令切换为50%Pn放电
    3. 记录从指令发出到功率达到-40%Pn的时间（反向穿越）
  
  判定：
    - 切换时间：≤200ms

【无功功率响应时间测试】
  方法：
    1. 在额定有功功率点
    2. 发指令从PF=1切换至PF=0.9（感性或容性）
    3. 记录无功功率达到90%目标的时间
  
  判定：
    - 响应时间：≤1s
```

### 7.3 容量测试

```
【测试目的】验证电池实际可用容量与标称值的偏差

【测试方法 - 标准容量测试法】
  1. 以C/3倍率(约0.33C)将电池充满至SOC=100%（按BMS显示的满载条件）
  2. 静置30min
  3. 以C/2倍率(约0.5C)恒功率放电至SOC=0%（按BMS显示的放电截止条件）
  4. 在直流侧使用功率分析仪积分计算放电总能量
  5. 记录环境温度（需保持25°C±2°C，如达不到需修正）

【容量计算】
  实测容量(C_meas) = ∫P_dc(t) dt × 1h / V_nom   (Ah)
  实测能量(E_meas) = ∫P_dc(t) dt                  (kWh)
  容量偏差 = (C_meas - C_rated) / C_rated × 100%
  能量偏差 = (E_meas - E_rated) / E_rated × 100%

【判定标准】
  - 电池容量偏差：实测≥标称值的100% (GB/T 36276)
  - 或按合同约定：实测≥标称值的95%为可接收

【温度修正】（如环境温度不是25°C）
  C_25°C = C_meas / [1 + α(T - 25)]
  其中 α = 温度系数（LFP约0.003/°C，三元约0.005/°C，以电池厂家数据为准）
```

### 7.4 SOC精度测试

```
【测试目的】验证BMS计算的SOC与实际SOC的偏差

【测试方法】
  1. 在PCS直流侧安装0.1级电流传感器+能量积分仪作为参考标准
  2. 执行一次完整充放电循环
  3. 以参考标准按安时积分法计算参考SOC：
     SOC_ref(t) = SOC_init + ∫I(t)dt × 100% / C_nom
  4. 在SOC=20%, 50%, 80%三个关键点比较BMS_SOC与SOC_ref

【判定标准】
  - SOC偏差：≤±5%（全SOC范围）
  - SOC=20%~80%区间：≤±3% (GB/T 34131)

【测试记录表】
  ┌──────────┬──────────┬──────────┬──────────┐
```

## 九、验收决策树

### 9.1 FAT放行决策树

```
所有A级（安全）项通过？
  ├── 否 → 禁止放行，必须整改
  │         └→ 整改完成后重新执行全部A级项测试
  │             ├── 通过 → 继续判断B级
  │             └── 仍不通过 → 启动退货/换货程序
  └── 是 → B级（功能/性能）项通过率？
            ├── 通过率≥95% → 可放行，记录余下NCR限期整改
            ├── 通过率80%~95% → 评审决策
            │   ├── 不影响安全和使用 → 有条件放行（附加条款）
            │   └── 影响现场安装/调试 → 不放行
            └── 通过率<80% → 不放行，全面整改
```

### 9.2 SAT并网决策树

```
安装检查通过？
  ├── 否 → 整改→复验
  └── 是
        ↓
    通信全链路打通且正确？
      ├── 否 → 排查通信问题
      └── 是
            ↓

          ├── 否 → 一票否决，不得并网
          │         └→ 重点：消防联动、防孤岛、急停
          └── 是
                ↓

              ├── 否 → 整改PCS/滤波措施，重测
              └── 是
                    ↓
                168h试运行通过？
                  ├── 否 → 修复故障后重新计时
                  └── 是 → 签发PAC → 进入商业运行
```

## 十、典型问题与处置

### 10.1 FAT常见问题

| 问题现象 | 可能原因 | 处置方法 |
|---------|---------|---------|
| 绝缘电阻测试不合格 | 潮湿、接线错误、绝缘损坏 | 烘干处理→检查接线→更换器件 |
| 耐压测试跳闸 | 绝缘距离不足、爬电距离不够 | 增加绝缘片→调整布局→返工 |
| BMS电芯电压全0 | 采集线接触不良、地址错误 | 重新插接采集线→配置BMU地址 |
| 通信无数据 | 波特率不一致、接线反、地址错 | 逐一核对通信参数→检查A/B线 |
| 保护不动作 | 定值错误、软件Bug | 重新核查定值→升级固件 |
| SOC跳变 | SOC初始值错误、电流传感器零漂 | 重新标定→校准电流传感器 |

### 10.2 SAT常见问题

| 问题现象 | 可能原因 | 处置方法 |
|---------|---------|---------|
| 并网电流谐波超标 | PCS滤波参数不匹配、电网背景谐波 | 调整PCS开关频率→增加无源滤波 |
| 防孤岛不动作 | RLC负载匹配不当、定值设置过高 | 精确调整RLC→核查保护定值 |
| 消防联动不走硬接线 | 依赖通信实现，无物理回路 | **严重！** 必须加装安全继电器→硬接线 |
| 遥信频繁抖动 | 信号干扰、接线松动、消抖不够 | 加屏蔽→紧固端子→软件增加消抖时间 |
| EMS与PCS功率偏差 | 电表CT变比配置错误、通信点表偏差 | 核查点表→校准CT→修正量程系数 |
| 电池温升异常 | 电池内阻偏高、冷却系统能力不足 | 检查冷却管道→确认电芯状态→降功率运行 |

### 10.3 应急处理流程

```
测试中发生异常事件
  │
  ├── 人身安全受威胁？ → 立即按下急停按钮 → 疏散人员 → 通知安全负责人
  │
  ├── 设备异常(异响/冒烟/泄漏)？
  │     ├── 立即停止测试
  │     ├── 断开所有电源（手动）
  │     ├── 如涉及电池热失控，启动消防预案
  │     ├── 记录事件时刻、参数快照
  │     └── 通知制造商技术支持
  │
  └── 测试数据异常(非安全类)？
        ├── 暂停当前测试项
        ├── 记录异常数据
        ├── 分析根因
        ├── 确认为测试方法问题→修正后继续
        └── 确认为设备问题→报NCR→等待处理
```
