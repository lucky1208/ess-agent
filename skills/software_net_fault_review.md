```
┌──────┬────────────────┬───────────────────────────────────────┐
│ 级别  │ 严重度          │ 描述与判定                            │
├──────┼────────────────┼───────────────────────────────────────┤

│      │ (Minor)        │ 如：单体温度偏高→降功率至70%            │
│      │                │ 24h内需处理，不处理可能升级到S3           │
├──────┼────────────────┼───────────────────────────────────────┤
│ S3    │ 中等/需人工介入  │ 设备/系统停机，但无安全风险              │
│      │ (Major)        │ 如：BCMU通信中断→该簇停机               │
│      │                │ 48h内需处理                              │
├──────┼────────────────┼───────────────────────────────────────┤
│ S4    │ 严重/紧急      │ 全系统/多设备停机，存在设备损坏风险       │
│      │ (Critical)    │ 如：多簇同时故障，PCS多模块故障          │
│      │                │ 需立即派人排除故障                       │
├──────┼────────────────┼───────────────────────────────────────┤
│ S5    │ 灾难/安全事故   │ 热失控/火灾/爆炸                       │
│      │ (Catastrophic) │ 如：电池热失控、内部电弧闪络              │
│      │                │ 需激活消防+全系统隔离+紧急响应           │
└──────┴────────────────┴───────────────────────────────────────┘
```

## 二、根因分析方法论

### 2.1 5-Why分析法 (储能电站定制版)

```
【5-Why法基本流程】
  1. 描述问题（What happened?）
  2. 第一问：为什么这个问题发生了？
  3. 第二问：为什么上一层的条件存在？
  4. ...持续5层
  5. 找到根因（不再追问"为什么"的最终原因）

【5-Why分析原则】
  - 每一层必须是"直接原因"（不能跳层）
  - 到第5层时应是"系统性根因"（过程/管理/设计问题）
  - 如第3层已到达物理/设计层面，可提前结束
  - 不要停在第1层"设备坏了"就停止追问

【示例：BMS上报"电芯电压偏高"告警】

  第1问：为什么电芯电压偏高？
    → BMS检测到第52串电芯电压3.75V，超过告警上限3.65V

  第2问：为什么第52串电芯电压达到3.75V？
    → PCS持续以额定功率充电，未响应BMS降功率请求

  第3问：为什么PCS未响应BMS降功率请求？
    → PCS控制器未收到BMS发来的CAN消息

  第4问：为什么PCS未收到BMS的CAN消息？
    → CAN通信因120Ω终端电阻缺失导致数据帧错误，PCS侧CAN控制器进入
      Bus-Off状态

  第5问：为什么120Ω终端电阻会缺失？
    → 出厂时未将CAN终端电阻跳线帽插入（FAT阶段未做CAN通信压力测试，
      测试规程未包含终端电阻检查，缺少防错检查清单）

  根因 → CAN终端电阻缺失 + FAT规程缺陷（系统性问题）
  
  纠正措施：
    - 立即：补装终端电阻，恢复BMS-PCS CAN通信
    - 短期：更新FAT检查清单，增列CAN终端电阻检查项
    - 长期：PCS侧增加"收不到BMS CAN帧超过5s则降功率"的软安全逻辑
```

### 2.2 鱼骨图 (Ishikawa) 分析法 — 储能专用分类

```
【储能故障鱼骨图分析框架】

                ┌── 设备/硬件 ──┐
                │ IGBT老化      │
                │ 电容衰减      │
                │ 连接松动      │
                │ 传感器漂移    │
                └──────────────┘
                
                ┌── 软件/固件 ──┐
                │ BMS参数漂移   │
                │ EMS策略Bug    │
                │ PCS版本缺陷   │
                │ 时序冲突      │
                └──────────────┘
                
                ┌── 通信/接线 ──┐
                │ CAN/485断线   │──────── 故
                │ 终端电阻缺失  │──────── 障
                │ 接地不良      │──────── 现
                │ IP冲突        │──────── 象
                └──────────────┘
                
                ┌── 环境/外部 ──┐
                │ 高温/高湿     │
                │ 雷击浪涌      │
                │ 电网波动      │
                │ 灰尘积聚      │
                └──────────────┘
                
                ┌── 人员/操作 ──┐
                │ 参数设置错误  │
                │ 维护操作不当  │
                │ 未按规程执行  │
                │ 培训不足      │
                └──────────────┘
                
                ┌── 管理/体系 ──┐
                │ 备件缺失      │
                │ 巡检遗漏      │
                │ 预案不完善    │
                │ 时间压力      │
                └──────────────┘

【使用流程】
  1. 召集相关方（运维/厂商/设计/管理）进行讨论
  2. 针对"故障现象"逐类找出所有可能原因（不求顺序，头脑风暴）
  3. 为每个原因分配可能性分数（1=低, 2=中, 3=高）
  4. 对可能性高的原因做验证测试
  5. 确定根因后标记到鱼骨图中
```

### 2.3 FMEA/FMECA方法

```
【FMEA (Failure Mode and Effects Analysis) 故障模式与影响分析】

RPN (Risk Priority Number, 风险优先级) = S × O × D

  S = 严重度 (Severity): 1~10分, 影响越大分越高
  O = 频度 (Occurrence): 1~10分, 发生越频繁越高
  D = 检测度 (Detection): 1~10分, 越难检测越高

RPN判定：
  RPN ≥ 100 (或≥150，根据组织定义) → 必须采取改进措施
  严重度 S ≥ 9 → 不论RPN多少，必须特殊关注（安全风险）

【储能FMEA片段示例】

┌──────┬────────────────┬───────┬───────┬───────┬───────┬──────────┐
│ 功能  │ 故障模式        │ S     │ O     │ D     │ RPN   │ 建议措施  │
├──────┼────────────────┼───────┼───────┼───────┼───────┼──────────┤
│ BMS  │ 电芯电压采样    │  8    │  3    │  4    │   96  │ 双通道    │
│ 采样  │ 偏差>20mV导致  │       │       │       │       │ 冗余采样  │
│      │ 均衡误动作      │       │       │       │       │           │
│      │                │       │       │       │       │           │
│      │ 电芯温度采样    │  7    │  4    │  3    │   84  │ 多点测温  │
│      │ 偏差>3°C       │       │       │       │       │ +定期校准 │
│      │                │       │       │       │       │           │
│ PCS  │ IGBT过温停机   │  6    │  5    │  2    │   60  │ 改善散热  │
│ 散热  │ 风扇转速不足   │       │       │       │       │ +风扇监控 │
│      │                │       │       │       │       │           │

└──────┴────────────────┴───────┴───────┴───────┴───────┴──────────┘
```

## 三、标准故障处置决策树

### 3.1 故障响应全局决策树

```
收到故障告警
    │
    ├── 告警等级？
    │   ├── S1 (信息) → 记录日志，纳入巡检
    │   ├── S2 (轻微) → 远程检查参数 → 是否需要降功率？
    │   │                ├── 是 → 远程降功率 → 安排计划维护
    │   │                └── 否 → 继续监控 → 计划处理
    │   │
    │   ├── S3 (中等) → 远程诊断
    │   │     ├── 能否远程恢复？ → 是 → 记录操作 → 跟踪监控
    │   │     └── 否 → 派人现场 → 排除故障 → 恢复正常
    │   │
    │   ├── S4 (严重) → 立即通知负责人
    │   │     ├── 执行远程安全停机（如适用）
    │   │     ├── 安排人员现场（2h内出发）
    │   │     ├── 现场处置 → 恢复运行
    │   │     └── 提交故障报告（48h内）
    │   │
    │   └── S5 (灾难) → 紧急响应
    │         ├── 立即电话通知所有相关方
    │         ├── 启动应急预案
    │         ├── 远程/人工紧急停机
    │         ├── 消防联动（如需要）
    │         ├── 事故调查（72h内启动）

### 4.2 电芯一致性劣化诊断流程

```
问题：BMS报"电芯压差异常"或"电芯容量偏差大"

【诊断步骤】

Step 1: 数据采集与分析
  - 导出现有全部电芯电压数据（至少24h）
  - 计算：均值、标准差、最大-最小差值
  - 绘制：时间序列图（标识问题电芯编号）
  - 如果所有电芯的V都呈一致性偏离 → SOC计算问题，非电芯问题

Step 2: 静态均衡功能检查
  - 将电池静置12h以上（SOC 30%~80%区间）
  - 检查均衡开关是否动作（查BMS均衡电流值）
  - 检查均衡电流是否在期望范围（通常50~300mA）
  - 如果均衡开关一直不动作 → BMS均衡策略问题

Step 3: 容量一致性测试
  - 执行C/3倍率一次完整充放电
  - 记录每个电芯在放电终止时的最低电压值
  - 如果用同一个放电电流，放电时间最短的电芯容量最低
  - 容量最低电芯 - 容量最高电芯 > 额定容量的5% → 异常

Step 4: 劣化判定
  如果单颗电芯(或少数几颗)的容量明显偏低且持续恶化 → "木桶效应"电芯
    ├── 如果均衡能够补偿 → 继续运行（降低充电上限，留出均衡空间）
    ├── 如果均衡无法补偿 → 计划停机，联系电池厂家更换模组/簇
    └── 如继续恶化，触发BMS过放保护之前应主动停用该簇

  如果是多颗电芯同步劣化 → 可能是系统性原因(高温/过流/生产批次问题)
    └── 需电池厂家全面分析
```

### 4.3 绝缘故障排查流程

```
BMS报"绝缘电阻过低"或"正极对地漏电流过大"

【排查步骤】

Step 1: 确认是否为真实绝缘故障
  - 断开电池系统与所有外部连接（PCS侧、BMS侧）
  - 用1000V兆欧表测量：
    - 电池正极总 → 外壳/大地：应≥10 MΩ
    - 电池负极总 → 外壳/大地：应≥10 MΩ
  - 如用兆欧表测也低 → 确实是电池系统内部绝缘问题

Step 2: 如果是真实绝缘故障 → 分段排查
  缩小范围法：
    a) 断开所有簇与直流母线的连接
    b) 逐一测量每个簇：正对地/负对地电阻
    c) 找到异常簇后，断开该簇内模组逐一测量
    d) 找到异常模组后，可能需拆开模组检查：
       - 电芯漏液/胀气/破损 → 更换模组
       - 采集线破损擦碰外壳 → 隔离修复
       - 水汽凝结 → 干燥处理

Step 3: 如果不是真实绝缘故障
  问题在BMS绝缘监测模块或检测算法：
    - 重新校准绝缘监测模块
    - 检查绝缘监测硬件电路（平衡电桥电路）
    - 更换故障模块

【安全提示】
  绝缘故障排查必须在：
  1. 断开交直流全部断路器
  2. 验电确认无电压
  3. 佩戴PPE(绝缘手套/绝缘鞋)
  后方可进行物理检查操作。
```

### 5.2 PCS功率模块故障深度分析

```
【IGBT模块失效模式分类和根因】

1. 过流/短路失效
   现象：模块短路，PCS保护跳闸
   原因：
     - 上下桥臂直通（死区时间不足或干扰导致的误导通）

     - 直流支撑电容击穿 → 放电冲击

     - 更换失效模块
     - 检查驱动板死区时间设定
     - 检查门极驱动电阻是否老化
     - 检查直流电容ESR/容量

2. 过压失效
   现象：集电极-发射极击穿
   原因：
     - 母线过压（电网侧暂态浪涌返回DC侧）
     - 开关瞬间感应过压（杂散电感过高）
   处置：
     - 更换模块
     - 检查母线DC过压保护是否正常
     - 降低直流母线额定电压运行（如果有裕量）

3. 过温失效
   现象：模块长期在高温下运行后退化
   原因：
     - 散热器与模块间导热硅脂干涸/缝隙
     - 风扇/液冷冷却能力不足
     - 频繁满载运行（或过载）
   处置：
     - 更换模块+重涂导热硅脂
     - 改善散热
     - 限制长期最大运行功率

4. 门极退化
   现象：开关特性变差，功耗增大
   原因：
     - 长期工作后门极氧化层退化
     - 制造缺陷
   处置：
     - 计划更换模块
```

            └── 附近有强干扰? → 加屏蔽/磁环→远离动力线

【RS485正常测量参数】

测量项              │ 正常范围             │ 异常解释
───────────────────┼──────────────────────┼──────────
A-B间电阻(断电)     │ 60Ω(2终端) / 120Ω(1终端) │ 断路/终端缺失

```

### 6.2 CAN总线故障诊断

```
【CAN点对点故障诊断】

CAN设备丢失/数据帧错误
    │
    ├── CAN_H/CAN_L线测量（断电）
    │   ├── CAN_H对GND电阻>500Ω? → 终端电阻正常（两端各120Ω）
    │   ├── CAN_H对CAN_L>万用表量程? → 线路完整无短路
    │   └── CAN_H对GND≈0Ω → 短路
    │
    ├── CAN总线终端电阻检查
    │   ├── 测CAN_H与CAN_L间电阻→应为60Ω(=120//120)
    │   │   ├── 是120Ω → 仅1个终端电阻（缺1个）
    │   │   └── 是40Ω → 3个终端电阻（多1个）
    │   │
    │   └── 带终端电阻检查（必须两端都有）
    │
    ├── 错误帧频发
    │   ├── 接地不良 → CAN_H/CAN_L对地压差大→接好地线
    │   ├── 支线(stub)过长 → 检查每节点支线长度
    │   │   └── 1Mbps：≤0.3m, 500kbps：≤0.5m, 250kbps：≤1m
    │   └── 终端电阻匹配错误 → 检查和校正
    │
    └── CAN控制器Bus-Off状态
        └── 累计错误次数超过255次 → CAN自动进入Bus-Off
            ├── 原因：连续帧错误(CRC/Ack/位错误)
            ├── 恢复：重新上电/Bus-Off Recovery软件自动恢复
            └── 预防：保证正确的终端电阻和接线质量

【CAN正常参数】

参数              │ 值               │ 
─────────────────┼──────────────────│
CAN_H-CAN_L电压差  │ 隐性≈0V, 显性≈2V │
终端电阻           │ 两端各120Ω       │
终端测总电阻       │ 60Ω (=120//120) │

### 6.3 以太网故障诊断

```
【以太网通信故障】

    │
    ├── 物理层检查

        │   └── 有RST包 → 连接被拒绝

### 7.2 防孤岛动作后的恢复流程

```
Step 1: 确认电网已恢复正常
  - 通过计量表和PCS面板确认并网点电压和频率在正常范围
  - 三相电压平衡，频率在49.5~50.5 Hz
  - 持续稳定至少3min以上（防电网闪变影响）

Step 2: 确认防孤岛装置已复位
  - 检查防孤岛保护装置状态：Stand By/Ready
  - PCC断路器已合闸待命

Step 3: 恢复BMS和PCS
  - 检查BMS无任何告警
  - 如因SOC保护停机，首先从电网充电恢复SOC
  - PCS执行软启动程序

Step 4: 验证并网
  - PCS并网运行，从0功率开始
  - 低功率运行10min，确认稳定
  - 逐步恢复调度功率指令

Step 5: 事件上报
  - 记录防孤岛动作时间、电网恢复时间
  - 分析动作原因（电网故障 vs 误动）
  - 如误动（电网正常时防孤岛误动作）→ 通知PCS厂家分析
```

### 8.2 热失控预警与响应

```
【热失控预警信号】

  Level 1 - N级预警（需密切关注）
    - 电芯温度上升速率 > 2°C/min
    - 单个电芯连续上升 >= 5°C 而其他正常
    - SOC异常消耗（无故放电）
    - 上述组合出现 → 进入Level 2

  Level 2 - 疑似热失控
    - 电芯温度持续上升 > 55°C
    - 该电芯电压异常下降
    - 该区域有异样气味
    → 立即降低该簇功率至0
    → 对周围电芯持续测温
    → 通知消防相关方
    → 如温度突破60°C → 进入Level 3

  Level 3 - 确认热失控
    - 电芯温度 > 65°C 且持续升高
    - 可燃气体检测(H2/CO)报警
    - 烟感/温感报警
    → 全系统紧急停机
    → 消防联动 → 全QF分断
    → 启动灭火装置
    → 疏散现场人员
    → 紧急报告 → 事故响应流程
```

### 9.2 基于故障模式的反向备件推荐

```
【故障模式→对应备件→推荐数量逻辑】

故障模式：BMS电芯电压采集异常
  → 可能涉及备件：BMU采集板(85%)、采集线缆(10%)、BCMU主板(5%)
  → 推荐备件：BMU板×1~2块（站点有N个BMU、MTTF=20,000h）
    公式：备件数 = ceil(N × 8760h/年 × 备件周期(年) / MTTF × 安全系数1.5)
    例如：20个BMU，补货周期0.5年，MTTF=30,000h
    备件数 = ceil(20 × 8760 × 0.5 / 30000 × 1.5) = ceil(4.38) = 5
    (实际中多数站点BMU板较可靠，采用1~2块即可)

故障模式：PCS IGBT模块故障
  → 可能涉及备件：功率模块(80%)、驱动板(15%)、DC电容(5%)
  → 推荐备件：功率模块×1（N-1冗余时）
    公式：模块备件数 = ceil(模块总数 × 8760 / MTBF × 补货年)
    例如：8个IGBT模块，MTBF=50,000h，补货期=0.1年
    备件数 = ceil(8 × 8760 × 0.1 / 50000) = ceil(0.14) = 1
```

## 十、RCM以可靠性为中心的维护决策

### 10.1 RCM决策逻辑框架

```
【储能RCM问题规范】

RCM-1: 功能失效 → 维护建议
  分析每个关键功能/设备的失效模式和影响
  为每种失效模式选择最合适的维护策略

RCM-2: 四种维护策略的决策树

  设备的重要性：
  失效会影响 → 安全/环境？ → 是 → CBM + RTF + 冗余设计
                       ↓
                     重大收益损失（停机>2h）？
                       ├── 是 → PdM (状态基) / CBM
                       └── 否
                            ↓
                         故障是可检测/可预测？
                           ├── 可预测 → CBM
                           ├── 可检测 → PM (定期)
                           └── 不可预测 → RTF (故障后修)
                                           └→但需要高MTBF，不能频繁

  策略词对照：
    PM = Preventive Maintenance (定期预防维护)
    CBM = Condition-Based Maintenance (状态基维护)
    PdM = Predictive Maintenance (预测性维护)
    RTF = Run to Failure (容故障运行（低影响时适用）)

【储能RCM示例】

设备        │ 失效模式     │ 影响       │ 策略    │ 维护措施
───────────┼─────────────┼───────────┼────────┼───────────
BMU采集    │ 电压偏差    │ 错误控制   │ CBM    │ 定期比对校准
风扇       │ 转速衰减    │ PCS降功率  │ RTF    │ 带备件运行
直流接触器  │ 主触烧损    │ 无法分合   │ CBM+PM │ 检测接触电阻+定期更换
PCS电容    │ ESR增大     │ 纹波增大   │ CBM    │ 每年检测ESR
BMS电池       │ 控制泄电  │ 时钟丢失   │ PM     │ 按3年周期更换
```

## 附录A：FMEA严重度/频度/检测度评分标准

```
【严重度 (S) 评分 (1~10)】

S10: 危险-可能导致人员伤亡或大面积环境损害（热失控/火灾）
S9:  可能造成严重设备损坏($XXX)或环境事故
S8:  全系统停车，无法恢复(>24h)
S7:  全系统停车，可恢复(4~24h)
S6:  部分系统停车(一簇/一PCS)，无法恢复(>6h)
S5:  部分系统停车(一簇/一PCS)，可恢复(2~6h)
S4:  降功率>50%，影响服务可用性(>4h)
S3:  降功率<50%，不影响基本服务
S2:  仅告警，可自动恢复
S1:  无影响

【频度 (O) 评分 (1~10)】

O10: 每天≥1次
O9:  每周≥1次(3~6次/月)
O8:  每月≥1次
O7:  每季度≥1次
O6:  每半年1次
O5:  每年1次
O4:  每2~5年1次
O3:  每5~15年1次
O2:  每15~20年1次(项目寿命期内1次)
O1:  非常罕见(>20年1次)

【检测度 (D) 评分 (1~10)】

D10: 不可检测(无检测手段/只能事后发现)
D9:  极难检测(需要拆解设备才能检测)
D8:  人工巡检可以发现，但无自动检测
D7:  有一定自动检测，但不够敏感
D6:  趋势分析或间接手段可以发现
D5:  有在线监测，但只在偏差大时发现
D4:  在线监测+趋势预警
D3:  在线监测+阈值告警(可预留时间处理)
D2:  冗余检测(多重检测互相校验)
D1:  确保可检测(双通道表决+高可靠性检测)
```
