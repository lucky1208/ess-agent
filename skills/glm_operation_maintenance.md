## 一、运维体系

### 1.1 运维体系架构

```
储能电站运维体系:
  ┌───────────────────────────────────────────────────┐
  │  日常巡检(每日/每班)                                │
  │    → 在线监测+现场巡视, 确保设备正常运行             │
  │                                                     │
  │  定期维护(月度/季度/年度)                            │
  │    → 按计划执行预防性维护, 防患于未然               │
  │                                                     │
  │  故障处理(随时)                                     │
  │    → 快速定位/隔离/修复/恢复, 最小化影响范围         │
  │                                                     │
  │  预防性维护(年度/大修)                              │
  │    → 深度检查/校验/更换, 延长设备寿命               │
  │                                                     │
  │  大修(3~5年)                                        │
  │    → 系统性检修/升级/改造                           │
  └───────────────────────────────────────────────────┘
  
  运维人员配置(5MW/10MWh电站):
    - 站长: 1人(全面管理)
    - 运维工程师: 2人(值班+巡检+维护)
    - 电气工程师: 1人(电气设备维护)
    - 合计: 4人(有人值守)
    - 或: 0人常驻+区域运维中心(无人值守)
```

### 2.3 电表与电网巡检

```
关口电表巡检(每日1次):
  - 正向有功电量(购电): 记录日用电量
  - 反向有功电量(售电): 记录日发电量
  - 无功电量: 记录无功补偿情况
  - 功率因数: ≥0.95(合同要求)
  - 最大需量: 记录月最大需量
  - 表计通信: 在线(采集系统)
  - 表计告警: 无失压/断流事件

电网参数巡检(实时):
  - PCC电压: 9.5~10.5kV
  - PCC频率: 49.8~50.2Hz
  - PCC功率: 记录充放电功率
  - 功率方向: 判断是否逆流
```

### 3.2 红外测温标准

```
红外测温判据(DL/T 664):
  电流致热型(接头/触头/线圈):
    δ = (T_1 - T_2) / T_2 × 100%
    T_1: 发热点温度, T_2: 正常参考温度
    
    ┌──────────┬──────────┬──────────┐
    │  δ值      │  状态     │  处理     │
    ├──────────┼──────────┼──────────┤
    │  <10%    │  正常     │  继续运行 │
    │  10~20%  │  一般缺陷 │  加强监测 │
    │  20~30%  │  严重缺陷 │  尽快处理 │
    │  >30%    │  危急缺陷 │  立即处理 │
    └──────────┴──────────┴──────────┘
  
  电压致热型(绝缘/避雷器):
    温差>5K: 一般缺陷
    温差>10K: 严重缺陷
    温差>15K: 危急缺陷
```

## 五、运维SOP模板

### 5.1 SOP标准格式

```
标准作业程序(SOP)模板:

═══════════════════════════════════════════════
  SOP编号: SOP-XX-001
  作业名称: PCS日常巡检标准作业程序
  适用范围: 所有PCS设备
  危险等级: 低(带电巡视, 不操作)
  作业人员: ≥1人(持电工证)
  工器具: 红外热像仪/万用表/对讲机/PPE
═══════════════════════════════════════════════

  作业前准备:
    □ 1. 穿戴PPE(安全帽/绝缘鞋/护目镜)
    □ 2. 携带工器具并检查完好
    □ 3. 确认PCS运行状态(EMS画面确认)
    □ 4. 通知值班人员巡检计划
  
  作业步骤:
    □ Step1: 外观检查
      - PCS柜门关闭, 无异响异味
      - 指示灯正常(运行绿灯亮, 告警灯灭)
      - 显示屏正常(无黑屏/花屏)
    
    □ Step2: 参数记录
      - 记录有功/无功功率
      - 记录交流侧电压/电流
      - 记录直流侧电压
      - 记录IGBT温度
      - 记录效率
    
    □ Step3: 红外测温
      - 测量进出线端子温度
      - 测量散热器温度
      - 判据: ΔT>10K记录, >20K告警
    
    □ Step4: 告警检查
      - 查看PCS告警列表
      - 确认无未处理告警
    
    □ Step5: 记录归档
      - 填写巡检记录表
      - 异常情况上报
  
  作业后:
    □ 1. 工器具清点归位
    □ 2. 巡检记录签字确认
    □ 3. 异常项进入缺陷管理流程

═══════════════════════════════════════════════
```

## 七、应急处理

### 7.1 火灾应急处置

```
火灾应急处置流程:
  发现火灾(烟感/温感/人工发现):
    │
    ▼
  Step1: 确认火情(视频监控/现场确认)
    │
    ▼
  Step2: 紧急停机
    - 按下急停按钮(就地)
    - 或EMS下发全站急停
    - 确认PCS停机+断路器跳闸
    │
    ▼
  Step3: 火灾报警
    - 确认消防主机启动
    - 拨打119(说明: 储能电站电池火灾+地址+联系方式)
    - 通知站长+业主
    │
    ▼
  Step4: 人员疏散
    - 启动声光报警
    - 按疏散指示撤离
    - 清点人数
    │
    ▼
  Step5: 等待消防(不可擅自灭火)
    - 电池火灾需专业消防处理
    - 不可用水直接扑救锂电池火灾
    - 保持安全距离(≥10m)
    │
    ▼
  Step6: 配合消防+事后处理
  
  关键原则:
    - 人身安全第一
    - 电池火灾严禁用水
    - 热失控可能复燃, 观察≥24h
```

### 7.2 BMS热失控应急处置

```
BMS热失控应急处置:
  触发条件: T>80℃ 或 dT/dt>1℃/s 或 烟感报警
  
  Step1: BMS自动急停
    - 断开故障簇接触器
    - 上报EMS热失控告警
  
  Step2: EMS全站停机
    - 所有PCS停机
    - PCC断路器跳闸
  
  Step3: 温控全力运行
    - Chiller/空调最大功率运行
    - 试图降低温度
  
  Step4: 消防联动
    - 消防信号触发
    - 气溶胶/七氟丙烷喷放
  
  Step5: 人员处置
    - 现场确认, 不可开舱门(氧气进入助燃)
    - 观察温度变化(红外热像仪隔门监测)
    - 温度持续上升→扩大隔离范围
  
  Step6: 后续处理
    - 确认安全后方可开舱检查
    - 故障电池簇隔离/更换
    - 事故调查+报告
```

PCS故障处置:

### 7.4 电网异常处置

```
电网异常处置:
  电压异常:
    - U>115%Un: PCS过压保护动作, 停机等待
    - U<85%Un: PCS欠压保护动作, 如能LVRT则支撑, 否则停机
    - 电压恢复后: PCS自动恢复(斜率≤10%Pn/s)
  
  频率异常:
    - f>51.5Hz: 过频保护动作, 停机
    - f<48.5Hz: 欠频保护动作, 停机
    - 49.5~50.5Hz: 一次调频响应
    - 频率恢复后: 自动恢复
  
  电网停电:
    - 孤岛检测→跳PCC断路器
    - 如有离网需求: 转离网运行(需配置)
    - 电网恢复: 检测电网正常→并网恢复
```

### 7.5 通信中断处置

```
通信中断处置:
  检测: 心跳超时(10s)
  
  影响范围:
    - EMS→PCS通信中断: PCS按最后指令运行(30s)→停机
    - EMS→BMS通信中断: BMS本地保护生效
    - EMS→调度通信中断: 调度视为离线, 可能切除AGC
    - 站内网络中断: 分区运行, 本地保护独立
  
  处置步骤:
    1. 确认中断范围(单设备/整站/调度)
    2. 检查物理连接(网线/光缆/交换机)
    3. 重启通信设备(网关/交换机)
    4. 检查IP地址/配置
    5. 通信恢复后确认数据一致
    6. 记录中断时间+影响+原因
    7. 如超30min未恢复→通知厂家
  
  通信冗余措施:
    - 双网: A/B网冗余
    - 双通道: 调度通信双路由
    - 边缘计算: 通信中断时本地策略执行
```

### 8.2 KPI计算代码

```python
def calculate_kpi(
    calendar_hours: float,
    running_hours: float,
    fault_count: int,
    total_fault_hours: float,
    alarm_count: int,
    alarm_respond_hours: list,
    charge_kwh: float,
    discharge_kwh: float,
    dod_actual: float,
    soc_cluster_std: float,
    defects_total: int,
    defects_resolved: int
):
    """运维KPI计算"""
    import statistics
    
    availability = running_hours / calendar_hours * 100 if calendar_hours > 0 else 0
    mtbf = running_hours / fault_count if fault_count > 0 else float('inf')
    mttr = total_fault_hours / fault_count if fault_count > 0 else 0
    avg_alarm_response = statistics.mean(alarm_respond_hours) if alarm_respond_hours else 0
    rt_efficiency = discharge_kwh / charge_kwh * 100 if charge_kwh > 0 else 0
    defect_resolution = defects_resolved / defects_total * 100 if defects_total > 0 else 100
    
    return {
        "系统可用率(%)": round(availability, 2),
        "可用率达标": availability >= 99,
        "MTBF(h)": round(mtbf, 1),
        "MTBF达标": mtbf >= 2000,
        "MTTR(h)": round(mttr, 2),
        "MTTR达标": mttr <= 4,
        "平均告警响应(min)": round(avg_alarm_response * 60, 1),
        "告警响应达标": avg_alarm_response * 60 <= 5,
        "充放电效率RT(%)": round(rt_efficiency, 2),
        "RT达标": rt_efficiency >= 90,
        "实际DOD(%)": round(dod_actual * 100, 1),
        "DOD达标": dod_actual >= 0.8,
        "簇间SOC偏差(%)": round(soc_cluster_std, 2),
        "一致性达标": soc_cluster_std <= 5,
        "缺陷消除率(%)": round(defect_resolution, 1),
        "缺陷消除达标": defect_resolution >= 90
    }

# 示例(月度KPI)
kpi = calculate_kpi(
    calendar_hours=720,
    running_hours=715,
    fault_count=2,
    total_fault_hours=3.5,
    alarm_count=15,
    alarm_respond_hours=[0.05, 0.08, 0.03, 0.12, 0.02, 0.04, 0.06, 0.03, 0.15, 0.02,
                          0.05, 0.04, 0.03, 0.08, 0.06],
    charge_kwh=500000,
    discharge_kwh=465000,
    dod_actual=0.88,
    soc_cluster_std=2.3,
    defects_total=8,
    defects_resolved=7
)
import json
print(json.dumps(kpi, ensure_ascii=False, indent=2))
```
