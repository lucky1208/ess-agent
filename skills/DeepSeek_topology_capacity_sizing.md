## 一、六大拓扑类型与决策树

### 1.1 拓扑分类体系

```
微网拓扑
├── 并网型微网
│   ├── Type A: 交流微网 (AC Microgrid)
│   │   └── 所有DER通过AC母线耦合，传统拓扑
│   ├── Type B: 直流微网 (DC Microgrid)
│   │   └── 所有DER通过DC母线耦合，适合直流负荷为主场景
│   ├── Type C: 交直流混合微网 (AC/DC Hybrid)
│   │   └── AC子网+DC子网通过双向变流器(ILC)耦合
│   └── Type D: 光储充一体化 (PV-BESS-EV Integrated)
│       └── 光伏→储能→充电桩全直流链路+交流并网
│
├── 离网型微网
│   ├── Type E: 柴储混合 (Diesel-BESS Hybrid)
│   │   └── 柴油发电机+储能作为主电源
│   └── Type F: 纯离网/孤岛 (Off-Grid/Island)
│       └── 完全脱离电网运行，需黑启动能力
│
└── 特殊型
    └── Type G: 级联型多微网 (Multi-Microgrid)
        └── 多个微网通过柔性互联或常规开关连接
```

### 1.2 6层拓扑决策树(完整版)

```
输入: 接入电网? ──否──→ 有柴发? ──是──→ 【Type E: 柴储混合】
    │                      │
    │                      └──否──→ 纯储能离网? ──是──→ 【Type F: 孤岛/黑启动】
    │
    ├── 是(并网型)
    │   │
    │   ├── 有充电桩?(≥5台, >300kW)
    │   │   ├── 是 ──→ 光伏+储能+充电桩? ──是──→ 【Type D: 光储充一体化】
    │   │   └── 否 ──→ 继续判断
    │   │
    │   ├── 直流负荷占比 > 60%?
    │   │   ├── 是 ──→ 交流负荷<10%? ──是──→ 【Type B: 直流微网】
    │   │   │         └──否──→ 【Type C: 交直流混合】
    │   │   └── 否 ──→ 继续判断
    │   │
    │   ├── 光伏DC/AC比 > 2.0?
    │   │   ├── 是 ──→ 建议DC侧直接耦合PCS ──→ 偏Type C或B
    │   │   └── 否 ──→ 继续判断
    │   │
    │   ├── 要求孤岛运行 + 黑启动?
    │   │   ├── 是 ──→ 需配置构网型PCS(GFM) ──→ 偏Type C(含GFM)
    │   │   └── 否 ──→ 跟网型PCS(GFL)即可
    │   │
    │   ├── 扩容预期高?(年增长>15%)
    │   │   ├── 是 ──→ 【Type C: 交直流混合】(扩展最灵活)
    │   │   └── 否 ──→ 继续判断
    │   │
    │   └── 预算极度紧张?
    │       ├── 是 ──→ 【Type A: 交流微网】(设备最成熟便宜)
    │       └── 否 ──→ 按综合评分最高选择 → 默认【Type C】
    │
    └── 特殊情况
        ├── 军用/海岛/极端偏远 → 考虑Type E或F
        ├── 多园区互联 → Type C级联 + 柔性互联
        └── 参与电力交易 → 需Type C(响应调度指令)
```

### 1.3 强制排除规则(一票否决)

| 条件 | 强制排除拓扑 | 原因 |
|------|------------|------|
| 无电网 > 6个月/年 | 排除A/B/C/D | 并网型无电网不可用 |
| 噪声要求 < 60dB且环境敏感 | 排除E | 柴油发电机噪声不达标 |
| 无柴油/燃气供应保障 | 排除E | 燃料不可得 |
| 海拔 > 3000m | E中柴发需降容 ≥ 30% | 高海拔需特殊选型 |
| 直流负荷 < 5% | 排除B(直流微网) | AC/DC转换不经济 |
| 预算 < 500万元 且 > 2MW | 排除C(先做A,后升级) | AC最经济 |
| 无LVRT/HVRT要求 | F可不考虑构网型 | 但功能受限 |
| 充电桩 < 3台且无光伏 | 排除D | 无光储充协同价值 |

## 四、交流微网拓扑详解

### 4.1 标准拓扑结构

```
                    ┌────────────────────────────┐
                    │     10kV/0.4kV 变压器      │
                    │     Dyn11, Uk%=6           │
                    └──────────┬─────────────────┘
                               │ 0.4kV AC Bus
              ┌────────────────┼────────────────┬──────────────┐
              │                │                │              │
         ┌────┴────┐     ┌────┴────┐     ┌────┴────┐    ┌────┴────┐
         │ 本地交流 │     │  PCS    │     │ 无功补偿 │    │  柴发   │
         │  负荷   │     │ 双向变流│     │  (SVG)  │    │ (备用)  │
         │ 400V    │     │ AC⇄DC  │     │  ±xkvar │    │  400V   │
         └─────────┘     └───┬──┬──┘     └─────────┘    └─────────┘
                             │  │
                          ┌──┘  └──┐
                     ┌────┴────┐  ┌┴────────┐
                     │ 光伏逆变 │  │  BESS   │
                     │ DC→AC   │  │ 电池舱  │
                     │ xxkWp   │  │ xxkWh   │
                     └─────────┘  └─────────┘
```

### 4.2 交流微网核心特性

| 特性 | 参数/描述 |
|------|----------|
| 耦合方式 | 共交流母线，所有设备并接到交流母线 |
| 光伏接入 | DC→AC通过组串/集中式逆变器并网 |
| 储能接入 | DC电池→PCS(AC/DC)→交流母线 |
| 变压器配置 | 10kV/0.4kV或10kV/0.69kV |
| 无功补偿 | 需单独配置SVG或电容柜 |
| 构网能力 | GFL(跟网型)为主，可配置GFM(构网型) |
| 标准化程度 | ★★★★★ (设备最成熟) |

## 七、光储充一体化拓扑详解

### 7.1 光储充三种子拓扑

| 子拓扑 | 光伏接入 | 储能接入 | 充电接入 | 效率 | 复杂度 |
|-------|---------|---------|---------|------|-------|
| AC耦合 | 逆变器→AC | PCS→AC | AC输入充电桩 | 低(多级) | 低 |
| DC耦合(集中) | DC/DC→DC Bus | DC/DC→DC Bus | DC Bus→分功率模块 | 高 | 中 |

## 十、储能容量优化算法(4种方法)

### 方法A:基于峰谷套利的迭代法

#### 核心公式

**每日峰谷套利收益**:
$$R_{daily} = \sum_{t=1}^{24} \left[ P_{dis}(t) - P_{ch}(t) \right] \times Price(t) \times \eta_{round-trip} \times \Delta t$$

**储能生命周期成本**:
$$C_{ESS} = C_{cap} + C_{om} \times L + C_{rep} - V_{salvage}$$

#### 迭代求解伪代码

```python
def iterative_search_irr(load_8760, tariff, P_range, E_range):
    """迭代搜索最优ESS配置(基于IRR最大化)"""
    results = []
    for P_ess in range(P_range[0], P_range[1]+1, P_range[2]):
        for E_ess in range(E_range[0], E_range[1]+1, E_range[2]):
            P_ch, P_dis, soc = optimize_dispatch(P_ess, E_ess, load_8760, tariff)
            R_annual = calc_annual_revenue(P_ch, P_dis, price_8760)
            capex_ess = 1500 * E_ess + 800 * P_ess
            opex_annual = 30 * E_ess
            annual_cf = R_annual - opex_annual
            NPV = -capex_ess + annual_cf * annuity_factor
            if annual_cf > 0:
                payback = capex_ess / annual_cf
                IRR_approx = 1/payback * lifetime
                results.append({'P_ess': P_ess, 'E_ess': E_ess, 'NPV': NPV, 'IRR': IRR_approx})
    return max(results, key=lambda x: x['IRR'])
```

### 方法B:基于风险调整收益最大化IRRVaR

**IRRVaR目标函数**:
$$\max_{P_{ess}, E_{ess}} \quad IRR_{expected} - \lambda \cdot CVaR_{\alpha}(IRR)$$

**风险偏好参数选择**:

| 风险类型 | λ取值 | 适用场景 |
|---------|-------|---------|
| 激进 | 0.1~0.2 | 电价政策稳定、补贴明确 |
| 中性 | 0.3~0.4 | 一般工商业项目 |
| 保守 | 0.5~0.7 | 政策不明朗、回收期敏感 |

### 方法C:基于MILP的两阶段优化

**决策变量**:

| 变量符号 | 变量名 | 类型 | 说明 |
|---------|--------|------|------|
| $P_{ch,t}$ | 充电功率 | 连续≥0 | 时段t充电功率(kW) |
| $P_{dis,t}$ | 放电功率 | 连续≥0 | 时段t放电功率(kW) |
| $SOC_t$ | 荷电状态 | 连续 | 时段t储能SOC(kWh) |
| $u_{ch,t}$ | 充电标志 | 二进制 | 充电=1 |
| $u_{dis,t}$ | 放电标志 | 二进制 | 放电=1 |

**目标函数**(日级运营成本最小化):
$$\min \sum_{t=1}^{96} \left[ P_{grid,t} \cdot Price_t \cdot \Delta t + u_{ch,t} \cdot C_{sw} \right]$$

### 方法D:基于强化学习的自适应容量规划

**状态空间**: 时段标志h + 光伏预测(24) + 负荷预测(24) + 当前SOC(1) + 电价预测(24)

**动作空间**: $\mathcal{A} = \{-1(放电), 0(待机), 1(充电)\}$

**奖励函数**:
$$R_t = (P_{dis,t} - P_{ch,t}) \cdot \lambda_t \cdot \eta_{RT} - C_{wear} \cdot |a_t|$$

**方法选择决策表**:

| 方法 | 适用场景 | 不适用场景 |
|------|---------|-----------|
| 方法A(迭代法) | 项目概算、初步选型 | 精确调度 |
| 方法B(IRRVaR) | 风险厌恶型投资者 | 极端价格波动 |
| 方法C(MILP) | 日级精确调度 | 求解器不可用 |
| 方法D(RL) | 现货市场、实时调度 | 数据训练成本高 |

## 十一、光伏容量匹配算法

### 11.1 自发自用率最优(负载匹配法)

**自发自用率**:
$$SCR = \frac{E_{self\_consumed}}{E_{PV\_total}} = \frac{\min(E_{PV}, E_{load})}{E_{PV}}$$

**消纳率约束下最大光伏**:
$$UR = \frac{E_{load\_total} - E_{grid\_export}}{E_{PV\_total}}$$

```python
def calc_self_consumption_ratio(P_pv_rated, load_8760, pv_8760):
    E_self = sum(min(pv_8760[t], load_8760[t]) for t in range(8760))
    E_pv_total = sum(pv_8760[t] for t in range(8760))
    SCR = E_self / E_pv_total if E_pv_total > 0 else 0
    return SCR
```

### 11.2 变压器容量约束

$$P_{grid\_max} = \sqrt{3} \times U_{grid} \times I_{max} \times \cos\phi$$

典型10kV并网点:
- 额定容量: $S_{trafo} = 2000 \text{kVA}$
- 最大允许电流: 115A
- 最大允许功率: 1732kW

### 14.2 液冷管路设计

#### 冷却液选型

| 冷却液类型 | 配方 | 冰点 | 沸点 | 适用场景 |

#### 流量计算公式

```

m = Q_cool / (Cp × ΔT)
```

#### 流量计算示例(2.5MW, 2h放电)

| 参数 | 数值 | 计算 |
|------|------|------|
| 电池内阻损耗 | ~1.5% | |
| 发热功率 | 2500kW×1.5%=37.5kW | |
| 冷却液比热容 | 3.5kJ/(kg·°C) | 40%乙二醇 |
| 进水/出水温度 | 10°C / 18°C | ΔT=8°C |
| 所需质量流量 | 37.5/(3.5×8)=1.34kg/s | |
| 体积流量 | 1.28L/s ≈ 77L/min | |

### 14.3 温控算法

**变流量温控(推荐)**:
```
PWM阀开度 = f(T_battery_avg, dT/dtime)
流量范围: 30%~100%额定流量
控制目标: T_battery_avg = 25~30°C
```

**温控参数目标**:

| 场景 | 进水温度 | 出水温度 | 舱内最高温度 | 温差 |
|------|---------|---------|------------|------|
| 标准工况 | 10~15°C | 15~22°C | ≤35°C | 5~8°C |
| 高温工况 | 15~20°C | 22~28°C | ≤40°C | 6~10°C |
| 低温启动 | 5~10°C | 10~18°C | ≤30°C | 3~5°C |

### 14.4 热失控防护设计

#### 三级防护体系

**第一级:预防**
- 电芯通过UL9540A测试
- BMS过温保护: >50°C降功率, >55°C停机
- 温度采样密度: 每个模组≥2个温度点
- 充电温控: <0°C限制充电≤0.1C

**第二级:控制**
- 模组间防火隔板: 耐火极限≥1h, 厚度≥50mm
- 模组级气溶胶灭火器: 触发温度70~80°C
- 簇级水喷淋: ≥5L/min·m²
- 舱内泄压口: ≥0.05m²

**第三级:灭火**
- 舱内水喷淋: ≥10L/min·m²
- 消防联动: 烟感/温感→BMS→PCS→消防主机
- 冷却时长: 热失控后持续喷淋≥30min

## 附录A:常用公式速查

```
电芯能量(Wh) = 额定容量(Ah) × 标称电压(V)
电芯功率(kW) = 容量(Ah) × 最大放电倍率(C) × 标称电压(V) / 1000
串联数S = ceil(PCS最高直流电压 / 电芯最高充电电压)
并联数P = ceil(PCS最大直流电流 / 电芯最大持续放电电流)
单簇容量(kWh) = S × P × 电芯额定容量(Ah) × 电芯标称电压(V) / 1000
电池发热功率(W) = 放电功率(kW) × (1/效率 - 1)
冷却液质量流量(kg/s) = 发热功率(W) / [比热容 × 温差]
```
