## 核心能力概览

本Skill定义储能电站EMS运营监控大屏的完整设计规范，涵盖：

- **四层数据展示架构**：从场站总览到设备详情到曲线分析到配置画面
- **实时监控画面设计**：核心数据、设备状态、告警信息流
- **能量流向可视化**：桑基图、功率箭头动画、SOC填充动画
- **告警管理与分级展示**：红/黄/蓝/灰四级告警、收敛规则
- **运营KPI仪表盘**：收益计算、能效指标、预测偏差分析
- **报表自动生成**：日报/月报/年报模板、电网调度报表、财务对账报表
- **技术实现建议**：前端选型、WebSocket架构、响应式设计

# 模块一：运营监控大屏设计规范

**L1层核心交互逻辑**：

| 交互动作 | 触发条件 | 响应结果 |
|---------|---------|---------|
| 点击功率仪表盘 | 单击/触屏tap | 弹出功率详情子画面（历史曲线、功率因数） |
| 点击SOC电池球 | 单击/触屏tap | 跳转L2层BMS详情画面 |
| 点击设备状态区 | 单击设备图标 | 跳转L2层对应设备详情（PCS/BMS/电表） |
| 点击功率曲线 | 单击时间点 | 跳转L3层该时刻历史数据查询 |
| 点击告警条目 | 单击告警行 | 展开告警详情，支持确认/处理操作 |

**L1层数据刷新机制**：
- 实时数据：WebSocket推送，刷新周期1秒
- 告警数据：WebSocket推送，新增告警即时闪烁
- 统计数据：5秒刷新一次（收益、充放电量累计）

### L4层：参数设置与策略配置（Configuration）

**定位**：仅授权运维人员可访问的参数配置界面，支持策略下发、设备参数调整。

**L4层安全控制**：
- 权限分级：操作员（只读）、运维（可调非安全参数）、管理员（全权限）
- 操作日志：所有修改记录审计日志，包含操作人/时间/修改前后值
- 确认机制：关键参数修改需二次确认
- 撤销机制：支持30分钟内撤销操作

**L4层配置分类**：

| 配置类别 | 包含内容 | 修改风险 |
|---------|---------|---------|
| BMS参数 | SOC告警阈值、温度限值、均衡策略 | 高（影响电池安全） |
| PCS参数 | 功率限值、功率因数、并离网参数 | 中（影响电网连接） |
| 策略参数 | 充放电时段、SOC目标、收益参数 | 中（影响运行策略） |
| 系统参数 | 通信超时、告警阈值、界面定制 | 低（系统配置） |

## 1.2 实时监控画面设计

### 1.2.1 核心数据展示模块

**设计原则**：
1. 数据优先级：功率>SOC>收益>充放电量
2. 刷新频率：关键数据1秒/次，次要数据5秒/次
3. 异常感知：数据越限自动变色（红=告警，黄=警告，绿=正常）
4. 历史趋势：小窗口显示最近1小时趋势（mini图）

**核心数据卡片规格**：

```yaml
# 核心数据卡片设计规范
data_card:
  type: "metric_card"  # 数据卡片类型
  layout:
    width: "flex"       # 宽度自适应
    min_width: "200px"
    max_width: "400px"
    height: "120px"
  
  data_visualization:
    primary_metric:
      type: "large_number"    # 大字号数字
      font_size: "36px"
      font_weight: "bold"
      unit: "kW / % / ¥"
      color: "dynamic"        # 根据数值动态变色
      
    secondary_metric:
      type: "trend_arrow"     # 趋势箭头
      indicator: "up/down/stable"
      comparison: "vs_yesterday"
      
    mini_chart:
      type: "sparkline"       # 小型趋势线
      data_points: 60         # 最近60个点
      height: "30px"
      color: "status_dependent"
```

### 1.2.3 告警信息流设计

**告警列表实时刷新规范**：

```yaml
# 告警列表设计规范
alarm_list:
  # 显示模式
  display_mode:
    type: "streaming_list"     # 流式列表（最新在上）
    max_visible: 10            # 可见告警数
    total_count: true          # 显示总数
    
  # 告警条目格式
  alarm_item:
    fields:
      - level_icon: "颜色圆点 + 级别文字"
      - equipment: "设备名称+编号"
      - message: "告警描述"
      - timestamp: "发生时间（相对/绝对）"
      - action: "操作按钮（确认/处理/详情）"
      
  # 告警聚合规则
  aggregation:
    enabled: true
    max_same_alarm_per_hour: 3   # 同类型告警每小时最多显示3条
    auto_hide_recovered: true     # 恢复告警自动隐藏（可查看历史）
    
  # 告警过滤
```

```yaml
# 收益计算公式定义
revenue_calculation:
  # 实时分钟收益
  minute_revenue:
    formula: |
      minute_revenue = current_power_kw × (1/60) × (current_price_yuan_per_kwh - charge_cost_yuan_per_kwh)
      
      其中：
      - 放电时：charge_cost = 0 （放电收益=功率×电价）
      - 充电时：revenue = -current_power_kw × (1/60) × charge_price_yuan_per_kwh / pcs_efficiency
      
    display:
      unit: "元/分钟"
      update_interval_sec: 60
      decimal_places: 2
      
  # 收益累计统计
  cumulative_revenue:
    today:
      formula: "Σminute_revenue (今日00:00至今)"
      display: "今日净收益: ¥X,XXX"
      
    this_month:
      formula: "Σdaily_revenue (本月1日至今)"
      display: "本月累计: ¥XXX,XXX"
      
    this_year:
      formula: "Σmonthly_revenue (本年1月至今)"
      display: "本年累计: ¥X,XXX,XXX"
      
  # 收益构成分析
  revenue_breakdown:
    discharge_revenue:
      formula: "Σ(discharge_kwh × discharge_price)"
      description: "放电售电收益"
      
    charge_cost:
      formula: "Σ(charge_kwh × charge_price) / pcs_round_trip_efficiency"
      description: "充电成本（含损耗）"
      
    demand_savings:
      formula: "(contract_demand - actual_max_demand) × demand_price_per_kw"
      description: "需量控制节省"
      
    auxiliary_cost:
      formula: "Σ(auxiliary_power_kw × duration_h × electricity_price)"
      description: "辅助系统耗电成本（空调、消防等）"
      
  # 净收益计算
  net_revenue:
    formula: |
      net_revenue = discharge_revenue - charge_cost + demand_savings - auxiliary_cost
    display:
      positive_color: "#52C41A"   # 绿色表示盈利
      negative_color: "#F5222D"   # 红色表示亏损
      
  # 收益预测
  revenue_forecast:
    today_forecast:
      method: "基于当前功率趋势外推"
      formula: "current_revenue + Σ(predicted_revenue_per_15min × remaining_slots)"
      
    monthly_forecast:
      method: "基于历史同期和当前趋势"
      confidence_interval: "±10%"
```

## 3.2 能效指标计算

### 3.2.1 核心能效指标公式

```yaml
# 能效指标计算公式
efficiency_metrics:
  # 充放电效率 (Round-Trip Efficiency, RTE)
  round_trip_efficiency:
    formula: |
      RTE = (total_discharge_kwh / total_charge_kwh) × 100%
      
      说明：
      - 包含PCS逆变损耗 + 整流损耗 + 电池内部损耗
      - 标准要求: RTE ≥ 85%（优质系统≥90%）
      
    display:
      unit: "%"
      normal_range: [85, 95]
      warning_threshold: 85
      optimal_target: 90
      
  # 电池可用容量
  battery_available_capacity:
    formula: |
      available_kwh = total_capacity_kwh × SOH × (SOC_upper - SOC_lower)
      
      说明：
      - SOC_upper: 策略运行上限（通常0.90-0.95）
      - SOC_lower: 策略运行下限（通常0.15-0.20）
      
    display:
      unit: "kWh"
      percentage_of_total: "显示占额定容量的百分比"
      
  # 自耗电率 (Self-Consumption Rate)
  self_consumption_rate:
    formula: |
      SCR = auxiliary_power_kwh / (charge_kwh + discharge_kwh) × 100%
      
      说明：
      - 辅助系统包括：空调、消防、照明、通信设备
      - 典型值: 1-3%（大型储能站），5-8%（小型储能）
      
    display:
      unit: "%"
      normal_range: [1, 5]
      
  # 系统可用率 (System Availability)
  system_availability:
    formula: |
      availability = available_time_h / total_time_h × 100%
      
      其中 available_time_h 计算方式:
      - 排除计划维护时间（可申请豁免）
      - 排除非计划停机时间（设备故障等）
      
    display:
      unit: "%"
      target: 98.0
      sla_threshold: 95.0
      
  # 等效循环次数 (Equivalent Full Cycles, EFC)
  equivalent_full_cycles:
    formula: |
      EFC = Σ(charge_discharge_kwh) / (2 × battery_capacity_kwh)
      
      说明：
      - 统计周期通常为月或年
      - 用于评估电池寿命消耗
      
    display:
      unit: "次/月" 或 "次/年"
      design_lifetime_cycles: 4000
      remaining_cycles_calculation: "design_lifetime - EFC"
      
  # 平均响应时间
  avg_response_time:
    formula: |
      avg_response = Σ(response_time_sec) / total_dispatch_count
      
      说明：
      - 响应时间 = 调度指令发出到PCS功率响应达90%的时间
      - 优质系统响应时间 < 1秒
      
    display:
      unit: "秒"
      target: < 2
```
