### 1.2 故障严重度分级(S1~S5)

| 级别 | 严重度 | 描述 | 处理时限 |
|------|--------|------|---------|
| S1 | 信息/提示 | 不影响运行(轻微通信延迟等), 自动恢复或无需干预 | 日志记录 |
| S2 | 轻微/预警 | 计划外降额/降功率运行, 24h内需处理 | 30min通知, 7天恢复 |
| S3 | 中等/需人工 | 设备/系统停机, 但无安全风险, 48h内需处理 | 15min通知, 72h恢复 |
| S4 | 严重/紧急 | 全系统/多设备停机, 存在设备损坏风险, 需立即派人 | 5min通知, 48h恢复 |
| S5 | 灾难/安全 | 热失控/火灾/爆炸, 需激活消防+全系统隔离+紧急响应 | 立即应急响应 |

| 级别 | 告警通知时限 | 运维响应时限 | 故障排除时限 | 备注 |
|------|-----------|-----------|-----------|------|
| S5 | 立即 | 15分钟(紧急) | 72小时 | 启动应急预案 |
| S4 | 5分钟 | 2小时 | 48小时 | 派人现场 |
| S3 | 15分钟 | 8小时 | 72小时 | 可远程先处理 |

## 二、根因分析方法论(5-Why+鱼骨图+FMEA)

### 2.1 5-Why分析法(储能电站定制版)

```text
【5-Why法基本流程】
  1. 描述问题（What happened?）
  2. 第一问：为什么这个问题发生了？
  3. 第二问：为什么上一层的条件存在？
  4. 持续5层（如第3层已到达物理/设计层面可提前结束）
  5. 找到根因（不再追问"为什么"的最终原因）

【示例：BMS上报"电芯电压偏高"告警】

  第1问：为什么电芯电压偏高？
    → BMS检测到第52串电芯电压3.75V, 超过告警上限3.65V

  第2问：为什么第52串电芯电压达到3.75V？
    → PCS持续以额定功率充电, 未响应BMS降功率请求

  第3问：为什么PCS未响应BMS降功率请求？
    → PCS控制器未收到BMS发来的CAN消息

  第4问：为什么PCS未收到BMS的CAN消息？
    → CAN通信因120Ω终端电阻缺失导致数据帧错误, PCS侧CAN控制器进入Bus-Off状态

  第5问：为什么120Ω终端电阻会缺失？
    → 出厂时未将CAN终端电阻跳线帽插入（FAT阶段未做CAN通信压力测试,
      测试规程未包含终端电阻检查）

  根因 → CAN终端电阻缺失 + FAT规程缺陷（系统性问题）

  纠正措施：
    - 立即：补装终端电阻
    - 短期：更新FAT检查清单, 增列CAN终端电阻检查项
    - 长期：PCS侧增加"收不到BMS CAN帧超过5s则降功率"的软安全逻辑
```

```text
                │ 雷击浪涌      │

                │ 参数设置错误  │

使用流程：
  1. 召集相关方进行讨论
  2. 逐类找出所有可能原因
  3. 为每个原因分配可能性分数(1~3)
  4. 对可能性高的原因做验证测试
  5. 确定根因后标记到鱼骨图中
```

### 2.3 FMEA/FMECA方法

```text
RPN (Risk Priority Number, 风险优先级) = S × O × D

  S = 严重度 (Severity): 1~10分
  O = 频度 (Occurrence): 1~10分
  D = 检测度 (Detection): 1~10分

RPN判定：
  RPN ≥ 100 → 必须采取改进措施
  严重度 S ≥ 9 → 不论RPN多少, 必须特殊关注（安全风险）

【储能FMEA示例】

┌──────┬────────────────┬───────┬───────┬───────┬───────┬──────────┐
│ 功能  │ 故障模式        │ S     │ O     │ D     │ RPN   │ 建议措施  │
├──────┼────────────────┼───────┼───────┼───────┼───────┼──────────┤
│ BMS  │ 电芯电压采样    │  8    │  3    │  4    │   96  │ 双通道    │
│ 采样  │ 偏差>20mV      │       │       │       │       │ 冗余采样  │
│ PCS  │ IGBT过温停机   │  6    │  5    │  2    │   60  │ 改善散热  │
│ 散热  │ 风扇转速不足   │       │       │       │       │ +风扇监控 │
│ EMS  │ 充电策略错误   │  8    │  2    │  3    │   48  │ 多条件校核│
│ 控制  │ SOC越限继续充电│       │       │       │       │ +保护切换 │
└──────┴────────────────┴───────┴───────┴───────┴───────┴──────────┘
```

## 四、BMS侧常见故障模式与处置

### 4.1 BMS故障模式分析表

| 故障模式 | 可能原因 | 严重度 | 处置建议 |
|---------|---------|--------|---------|
| 个别电芯电压为0(无读数) | BMU采集线断或BMU掉线 | S3 | 检查采集线→重插→如不恢复更换BMU |
| 电芯电压读数明显偏高/低(固定偏差) | BMU采样通道漂移 | S3 | 用万用表实测→对比→偏差大校准/更换BMU |
| 温度读数异常(-40/85°C等) | NTC线断或短路 | S3 | 检查NTC探头→连线→更换传感器 |
| 所有电芯电压剧烈波动 | BMU地线浮空/干扰 | S4 | 检查BMS参考地→加固接地→加屏蔽 |
| 绝缘电阻检测值异常跳变 | 绝缘检测模块故障/接线 | S3 | 分离电池和BMS, 用专用表实测绝缘 |
| SOC快速跳变(不连续) | 电流传感器零漂大 | S2 | 用精密源校准→如零漂超限更换 |
| BMU通信中断 | 线缆松动/BMU死机 | S3 | 检查接头→重插→重启→更換缆线 |
| 接触器不吸合/不释放 | 接触器线圈或BMS输出故障 | S4 | 测线圈电压→查BMS输出继电器→更换 |
| 接触器粘连(无法分断) | 接触器主触点熔焊 | S5 | **立即停机**→隔离电池→更换接触器 |

### 4.2 电芯一致性劣化诊断流程

```text
问题：BMS报"电芯压差异常"或"电芯容量偏差大"

Step 1: 数据采集与分析
  - 导出全部电芯电压数据（至少24h）
  - 计算：均值、标准差、最大-最小差值
  - 所有电芯V一致性偏离 → SOC计算问题, 非电芯问题

Step 2: 静态均衡功能检查
  - 将电池静置12h以上（SOC 30%~80%区间）
  - 检查均衡开关动作及均衡电流值(50~300mA)
  - 均衡开关一直不动作 → BMS均衡策略问题

Step 3: 容量一致性测试
  - 执行C/3倍率一次完整充放电
  - 容量最低-最高 > 额定容量5% → 异常

Step 4: 劣化判定
  单颗电芯容量明显偏低且持续恶化 → "木桶效应"电芯
    ├── 均衡能补偿 → 继续运行（降低充电上限）
    ├── 均衡无法补偿 → 计划停机，联系厂家更换模组/簇
    └── 继续恶化 → 触发过放保护前主动停用该簇
```

### 4.3 绝缘故障排查流程

```text
BMS报"绝缘电阻过低"

Step 1: 确认是否为真实绝缘故障
  - 断开电池系统与所有外部连接
  - 用1000V兆欧表测量：正极总对地 ≥10 MΩ, 负极总对地 ≥10 MΩ

Step 2: 真实故障 → 分段排查缩小范围法
  a) 断开所有簇与直流母线连接
  b) 逐一测量每个簇：正对地/负对地电阻

  - 重新校准绝缘监测模块
  - 检查绝缘监测硬件电路

安全提示：必须在断开全部断路器、验电确认无电压、佩戴PPE后方可操作
```

## 九、SOH估算与寿命预测算法

### 9.1 安时积分法(Ah Counting)

```python
import numpy as np

def estimate_soh_ah_counting(df_charge, df_discharge, C_nom,
                              calibration_interval_h=168, initial_soh=1.0):
    """
    安时积分法估算SOH
    原理：通过精确积分充放电电流, 计算实际转移的电荷量
    优点：实现简单、计算量小、可在线运行
    缺点：存在积分漂移累积误差, 需定期标准容量校准
    """
    df = pd.concat([df_charge, df_discharge]).sort_values('timestamp')
    df['dt'] = df['timestamp'].diff().dt.total_seconds() / 3600.0
    df['dQ'] = df['current'] * df['dt']

    total_charge = df[df['current'] > 0]['dQ'].sum()
    total_discharge = abs(df[df['current'] < 0]['dQ'].sum())
    Q_actual = min(total_charge, total_discharge)
    soh = Q_actual / C_nom if C_nom > 0 else initial_soh
    return round(np.clip(soh, 0.0, 1.0), 4)
```

### 9.2 扩展卡尔曼滤波(EKF) — 工业级在线SOH估计

```python
import numpy as np

class ExtendedKalmanFilterSOH:
    """基于扩展卡尔曼滤波的SOC+SOH联合估计器
    状态向量: x = [SOC, SOH]^T
    观测向量: y = V_terminal

    原理：使用电池等效电路模型(ECM), 通过预测（基于电池模型）
    和更新（基于实测电压）两步递归, 抑制积分漂移和测量噪声
    优点：实时最优估计、抗噪声强、无需定期校准、可同时估计SOC和SOH
    """

    def __init__(self, dt=1.0, temperature=25.0):
        self.dt = dt
        self.temperature = temperature
        self.x = np.array([[0.80], [1.00]])   # [SOC, SOH]
        self.P = np.diag([0.01, 0.01])        # 协方差矩阵
        self.Q = np.diag([0.0001, 0.00001])   # 过程噪声
        self.R = np.diag([0.01])               # 观测噪声
        self.model_params = self._get_params(temperature)

    def _get_params(self, T):
        return {
            'R0_25': 0.0012, 'R0_T': 0.00002,
            'R1_25': 0.0025, 'C1_25': 3000.0,
            'V_full': 3.40, 'V_empty': 2.80, 'Q_nom': 280.0
        }

    def predict(self, I):
        """EKF预测步：状态预测 + 协方差预测"""
        dt = self.dt
        SOC, SOH = float(self.x[0]), float(self.x[1])
        Q_nom = self.model_params['Q_nom']
        SOC_new = SOC - (I * dt / 3600.0) / (Q_nom * SOH)
        SOC_new = np.clip(SOC_new, 0.0, 1.0)
        self.x = np.array([[SOC_new], [SOH]])
        F = np.array([[1, I * dt / 3600.0 / (Q_nom * SOH ** 2)], [0, 1]])
        self.P = F @ self.P @ F.T + self.Q

    def update(self, V_measured, I):
        """EKF更新步：利用观测值修正状态"""
        SOC = float(self.x[0])
        params = self.model_params
        V_ocv = params['V_empty'] + (params['V_full'] - params['V_empty']) * SOC
        dV_dSOC = params['V_full'] - params['V_empty']
        y_innovation = V_measured - V_ocv
        H_simple = np.array([[dV_dSOC, 0.0]])
        S = H_simple @ self.P @ H_simple.T + self.R
        K = self.P @ H_simple.T / float(S)
        self.x = self.x + K * y_innovation
        self.x[0] = np.clip(self.x[0], 0.0, 1.0)
        self.x[1] = np.clip(self.x[1], 0.0, 1.0)
        I_mat = np.eye(2)
        self.P = (I_mat - K @ H_simple) @ self.P

    def step(self, V_measured, I):
        """执行一次EKF迭代（预测+更新）"""
        self.predict(I)
        self.update(V_measured, I)
        return float(self.x[0]), float(self.x[1])
```

### 9.3 OCV开路电压法

```text
原理：锂电池的开路电压(OCV)与SOC有明确对应关系(通过标准OCV-SOC曲线)
当电池静置足够长时间后(通常2h以上), 测量OCV即可推算当前SOC

LFP典型OCV-SOC曲线（需根据实际电芯型号标定）：
  (2.80V, 0%), (3.00V, 2%), (3.20V, 10%), (3.30V, 20%)
  (3.34V, 40%), (3.36V, 50%), (3.38V, 60%), (3.40V, 70%)
  (3.45V, 90%), (3.50V, 98%), (3.60V, 100%)

优点：精度高、无需电流积分、不需要额外传感器
缺点：需要电池充分静置, 运行时无法测量

适用场景：月度/季度标准容量测试时的精确SOH标定
```

### 9.4 放电曲线特征提取法

```text
原理：在标准放电测试中, 通过分析放电电压曲线的特征变化
(压降斜率、拐点位置)来判断SOH, 是工业现场最实用的方法

特征提取指标：
  1. 放电中点电压(V_mid)：放出50%容量时的电压
  2. 放电末端电压跌落斜率(V_tail_slope)：最后10%容量的电压下降速率
  3. 等压平台时长(plateu_duration)：电压在3.2~3.4V(LFP平台区)的持续时间
  4. 电压拐点位置(拐点_capacity_ratio)：dV/dt最小点位置

优点：不需要模型参数, 对电芯老化敏感
缺点：需要标准放电测试, 深放电影响收益

SOH估算 = Σ(各特征当前值/基准值 × 权重)
  推荐权重：V_mid=40%, V_tail_slope=20%, plateu_duration=20%, 拐点=20%
```

## 十、关键器件剩余寿命预警

### 10.1 PCS IGBT剩余寿命 — LESIT加速模型

```python
import numpy as np

def estimate_igbt_remaining_life(total_runtime_h, avg_switching_cycles_per_hour=300,
                                  ΔT_j=40.0, T_j_max=125.0, T_j_mean=85.0):
    """
    IGBT剩余寿命估算（基于LESIT/CIPS2008加速寿命模型）

    公式: N_f = A × (ΔT_j)^(-3.3) × exp(Ea/(R×T_j_mean_K))

    参数:
        total_runtime_h: 累计运行小时
        ΔT_j: 结温波动幅度(°C)
        T_j_max: 最大结温(°C)
        T_j_mean: 平均结温(°C)

    IGBT主要老化机理：功率循环导致焊料层疲劳和绑定线退化
    老化程度与结温波动幅度ΔTj和循环次数直接相关
    """
    A = 9.45e13          # LESIT模型常数
    Ea = 7.77e4          # 激活能 J/mol
    R = 8.314            # 气体常数

    T_j_mean_K = T_j_mean + 273.15
    N_f = A * (ΔT_j ** -3.3) * np.exp(Ea / (R * T_j_mean_K))

    total_switching_cycles = total_runtime_h * avg_switching_cycles_per_hour
    consumed_ratio = min(1.0, total_switching_cycles / N_f)
    remaining_cycles = max(0, N_f - total_switching_cycles)
    remaining_hours = remaining_cycles / avg_switching_cycles_per_hour

    # 剩余寿命预警等级
    if consumed_ratio >= 0.90:
        level = 'CRITICAL'
    elif consumed_ratio >= 0.75:
        level = 'WARNING'
    elif consumed_ratio >= 0.60:
        level = 'CAUTION'
    else:
        level = 'NORMAL'

    return {
        'igbt_cycles_to_failure': round(N_f, 0),
        'consumed_lifetime_ratio': round(consumed_ratio, 4),
        'remaining_hours': round(remaining_hours, 0),
        'remaining_years': round(remaining_hours / 8760, 1),
        'alert_level': level
    }
```

### 10.2 PCS电容剩余寿命 — Arrhenius加速模型

```python
def estimate_capacitor_remaining_life(total_runtime_h, I_ripple_rms,
                                       C_rated, ESR_rated, T_ambient,
                                       T_cap_max=105.0, L_target=50000.0):
    """
    电容剩余寿命估算（基于Arrhenius加速模型）

    原理：纹波电流I_ripple导致电容内部温升
         核心：ΔT = I_ripple² × ESR / (k × A)

    Arrhenius温度加速：
    AF = exp(Ea/R × (1/(T_cap_max+273.15) - 1/(T_hotspot+273.15)))
    """
    k_thermal = 0.01
    A_surface = 0.01
    Ea = 80000     # 电容降解激活能 J/mol
    R = 8.314

    # 纹波电流导致的内部温升
    ΔT = (I_ripple_rms ** 2) * ESR_rated / (k_thermal * A_surface) * 1000
    T_hotspot = T_ambient + ΔT

    # Arrhenius温度加速因子
    AF = np.exp(Ea / R * (1/(T_cap_max + 273.15) - 1/(T_hotspot + 273.15)))

    L_remaining = L_target / AF if AF > 0 else float('inf')

    return {
        'T_hotspot_c': round(T_hotspot, 1),
        'ΔT_c': round(ΔT, 1),
        'acceleration_factor': round(AF, 2),
        'remaining_lifetime_h': round(L_remaining, 0),
        'remaining_lifetime_years': round(L_remaining / 8760, 1),
        'alert_level': 'WARNING' if L_remaining < 20000 else 'NORMAL'
    }
```

### 10.3 Arrhenius温度加速因子(电池通用)

```python
def arrhenius_acceleration_factor(T_op, T_ref=25.0, Ea=20000.0, R=8.314):
    """
    计算温度加速因子（Arrhenius方程）

    AF = exp(Ea/R × (1/T_ref_K - 1/T_op_K))

    锂离子电池典型Ea值：18000~22000 J/mol
    """
    T_ref_K = T_ref + 273.15
    T_op_K = T_op + 273.15
    exponent = (Ea / R) * (1.0 / T_ref_K - 1.0 / T_op_K)
    return float(np.exp(exponent))
```

## 十一、电池衰减预测与退化管理

### 11.1 年衰减率趋势外推(线性回归)

```python
from scipy import stats

def predict_soh_linear_regression(soh_history, forecast_years=5, confidence=0.95):
    """
    基于历史SOH数据的线性回归预测
    返回: 年衰减率、各年预测SOH、置信区间、EOL日期、R²
    """
    df = soh_history.copy()
    df['days'] = (df['date'] - df['date'].min()).dt.days
    df['year'] = df['days'] / 365.25

    slope, intercept, r_value, p_value, std_err = stats.linregress(
        df['year'], df['soh']
    )

    annual_decay_rate = abs(slope)
    residuals = df['soh'] - (intercept + slope * df['year'])
    residual_std = np.std(residuals)
    t_value = stats.t.ppf((1 + confidence) / 2, len(df) - 2)
    margin = t_value * residual_std

    # EOL预测（SOH降至80%）
    eol_years = (0.80 - intercept) / slope if slope != 0 else None

    return {
        'annual_decay_rate': round(annual_decay_rate, 4),
        'R2': round(r_value ** 2, 4),
        'eol_years_from_now': round(eol_years, 1) if eol_years else None
    }
```

### 11.2 等效循环次数统计

```python
def calculate_equivalent_cycles(df_operation):
    """
    DOD权重公式: eq_cycle_factor = (DOD / 100) ^ 1.5
    100%DOD等效1次；80%DOD等效0.72次；50%DOD等效0.35次
    """
    df = df_operation.copy()
    df['eq_factor'] = (df['dod_pct'] / 100.0) ** 1.5
    df['eq_cycles'] = df['eq_factor']

    total_eq = df['eq_cycles'].sum()
    total_real = len(df)
    return {
        'total_equivalent_cycles': round(total_eq, 2),
        'total_real_cycles': total_real
    }
```

### 13.3 基于故障率的备件消耗预测

```text
公式：备件数 = ceil(N × 8760h/年 × 备件周期(年) / MTTF × 安全系数1.5)

示例：20个BMU, 补货周期0.5年, MTTF=30,000h
  备件数 = ceil(20 × 8760 × 0.5 / 30000 × 1.5) = ceil(4.38) = 5
  (实际中多数站点BMU板较可靠, 采用1~2块即可)

推荐使用泊松分布建模稀有事件的故障次数:
  安全库存 SS = Poisson_inv(service_level, λ × forecast_period)
```
